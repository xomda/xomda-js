# Support

Need help with xomda? Here is where to look.

## Documentation

Start with the docs — most questions are answered there:

- [Getting started](./docs/getting-started.md) — first project, first model, first generation.
- [Concepts](./docs/concepts.md) — the ideas behind xomda.
- [Data model](./docs/data-model.md) — schema reference.
- [Templates](./docs/templates.md) — the template language.
- [API](./docs/api.md) — tRPC API reference.
- [Architecture](./docs/architecture.md) — codebase layout.
- [Development](./docs/development.md) — local setup and troubleshooting.

The full index is in [`docs/README.md`](./docs/README.md).

## Questions and discussions

For general questions, ideas, or design discussions, use
[GitHub Discussions](https://github.com/JorisAerts/modelman/discussions) (if enabled) or open an issue tagged
`question`.

## Bugs and feature requests

- **Bugs** — open an issue using the [bug report template](https://github.com/JorisAerts/modelman/issues/new?template=bug_report.yml).
- **Feature requests** — open an issue using the [feature request template](https://github.com/JorisAerts/modelman/issues/new?template=feature_request.yml).

Please search existing issues first to avoid duplicates.

## Security

For security-sensitive reports, follow the process in [SECURITY.md](./SECURITY.md) — do not open a public issue.

## Contributing

If you would like to contribute a fix or a feature yourself, see [CONTRIBUTING.md](./CONTRIBUTING.md).
